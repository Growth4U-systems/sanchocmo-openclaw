## ✅ MC Chat reescrito — Gateway WebSocket directo (00:26+)

**Decisión**: Abandonar el plugin mc-chat. El chat de Mission Control ahora se conecta directamente al Gateway WebSocket, igual que hace el Control UI nativo de OpenClaw.

**Qué cambió**:
- Chat column de MC reescrito en `mission-control.html`
- Eliminado: JSON files, polling, `openclaw agent` CLI, endpoints `/api/chat/*` (quedan en mc-server pero frontend no los usa)
- Añadido: Cliente WS directo (`chat.send`, `chat.history`, eventos `chat` para streaming)
- Auth: lee token de Control UI localStorage, sessionStorage, o URL hash
- Streaming real: mensajes aparecen caracter a caracter
- Session selector: puede cambiar entre sesiones del Gateway
- Auto-reconnect con backoff de 5s
- Plugin mc-chat movido de `~/.openclaw/extensions/` a `workspace-cervantes/plugins-wip/` para evitar auto-discovery

**Arquitectura**:
```
MC HTML → WebSocket → Gateway (:18789) → Sancho agent
                  (mismo protocolo que Control UI)
```

**Ventajas vs plugin**:
- Zero plugins, zero middleware, zero JSON files
- Streaming real (no polling cada 2s)
- Misma infra de auth que Control UI (Tailscale, token, password)
- Sessions del Gateway reales (historial, abort, etc.)
