**Lo que se aprendió del proxy WS (útil para el futuro)**:
- Gateway protocol: `connect.challenge` → Ed25519 device signature → `connect` → `hello-ok`
- Valid client.id: `cli`, `webchat`, `openclaw-control-ui`, `gateway-client`, etc.
- Valid client.mode: `webchat`, `cli`, `ui`, `backend`, `node`, `probe`, `test`
- Mode `webchat` NO concede `operator.write` — necesita device auth completo
- `dangerouslyDisableDeviceAuth` no concede write scopes, solo salta identity check
- Device auth: Ed25519 keypair, sign v2 payload (`v2|deviceId|clientId|mode|role|scopes|ts|token|nonce`)
- Localhost auto-approves pairing pero SOLO después de un `openclaw devices approve`
- Config `allowInsecureAuth` es para HTTP non-secure context, no para scope elevation

## 🧠 Learnings clave de esta sesión (para MEMORY.md en próxima revisión)

### Gateway WS Protocol (nuevo learning profundo)
- **connect.challenge** es obligatorio — no se puede saltar
- **Device auth Ed25519** es requerido para `operator.write` scopes — sin firma, solo read
- v2 payload: `v2|deviceId|clientId|clientMode|role|scopes|signedAt|token|nonce`
- Nuevo device → requiere `openclaw devices approve <requestId>` incluso en localhost
- Mode `webchat` tiene scopes limitados por diseño — para write necesitas `cli` o `openclaw-control-ui` con device auth completo
- `dangerouslyDisableDeviceAuth` salta el identity check pero NO eleva scopes
- `allowInsecureAuth` es para HTTP non-secure context, no para scope elevation

### Error de approach: WS proxy vs Channel plugin
- **Pasé 5+ horas** construyendo un proxy WS que reimplementa lo que el Channel system ya hace
- El plugin mc-chat original tenía la arquitectura correcta desde el principio
- **Learning**: Antes de reimplementar desde cero, verificar si el componente existente tiene un bug puntual vs un defecto de diseño. En este caso era un bug puntual (`listAccountIds` faltaba) — el diseño del plugin era correcto
- **Learning**: Alfonso tiene razón sobre usar Channels — es portable, funciona hosteado, y es el patrón estándar de OpenClaw. El proxy WS es un hack que solo funciona en localhost/Tailscale
