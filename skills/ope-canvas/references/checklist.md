# OPE Canvas — Self-QA Checklist

## Completitud
- [ ] Las 14 secciones están presentes
- [ ] Obvious Choice tiene 2-3 puntos (no features)
- [ ] ICP tiene cualificadores Y descalificadores
- [ ] Core Problem separa funcional de emocional
- [ ] Core Product tiene 2-4 componentes (no feature dump)
- [ ] Geography tiene fases (actual + expansión)
- [ ] Channels son top 3-4 con propósito estratégico
- [ ] 3 Moats clasificados por tipo según framework
- [ ] Endgame tiene declaración cualitativa + métrica cuantitativa
- [ ] Core Values son 3-5 (no inventados)
- [ ] Strategy Choice es coherente con Canales + ICP + Product
- [ ] Monthly Picture tiene datos reales (o DUDA justificada)

## Calidad
- [ ] Toda la información viene de docs del cliente (nada inventado)
- [ ] Cada DUDA está explícita y justificada
- [ ] Cabe conceptualmente en una página (conciso)
- [ ] No hay moats genéricos ("buen equipo", "tecnología puntera")
- [ ] ICP no es demasiado amplio
- [ ] Endgame tiene métrica, no solo deseo

## Coherencia
- [ ] Strategy Choice coherente con Channels elegidos
- [ ] Moats soportan el Obvious Choice
- [ ] Core Capabilities habilitan los Moats
- [ ] Quarterly Picture avanza hacia Year Picture
- [ ] Year Picture avanza hacia Endgame

## Fuentes
- [ ] Datos cruzados contra company-context y business-model
- [ ] DUDAs son preguntas para Power Hour (no gaps ignorados)
