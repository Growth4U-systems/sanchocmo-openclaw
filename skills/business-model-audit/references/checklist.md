# Business Model & Growth Model — Self-QA Checklist

> El agente DEBE revisar este checklist ANTES de entregar el documento.
> Para cada ítem: ✅ completado | ⚠️ no disponible (con justificación) | ❌ pendiente (seguir investigando)
> Solo se entrega cuando todo es ✅ o ⚠️. Si hay algún ❌, volver a investigar.

---

## Step 1: Clasificación del Modelo

- [ ] **Modelo de revenue identificado** (suscripción, transacción, marketplace, freemium, pago único, hybrid)
- [ ] **Primary + secondary revenue** diferenciados (si aplica)
- [ ] **Ticket medio** capturado (o Discovery Task asignado)
- [ ] **LTV estimado** (calculado o Discovery Task asignado)
- [ ] **Revenue expansion vs nuevos clientes** documentado (o Discovery Task)
- [ ] **Arquetipo mapeado** a tabla de modelos (ACV, motion natural, canal primario, warning sign)
- [ ] **FinTech constraints** aplicados (si aplica: regulación, trust signals, compliance)

## Step 2: Growth Motion

- [ ] **PLG/MLG/Sales-Led/Hybrid** clasificado con evidencia
- [ ] **Self-serve capability** evaluada (¿puede empezar solo?)
- [ ] **Fuentes de adquisición actuales** mapeadas (de dónde vienen clientes)
- [ ] **Comparación motion vs competidores** realizada (si competitor-intel disponible)
- [ ] **Misalignment detectado** (si cliente cree ser PLG pero no lo es → señalado)

## Step 3: Funnel Actual

- [ ] **Camino completo mapeado** (descubrimiento → pago, paso a paso)
- [ ] **Bottleneck identificado** (dónde se pierden más personas)
- [ ] **Conversión por paso** documentada (measured / estimated / unknown)
- [ ] **Cada paso unknown** → Discovery Task asignado

## Discovery Tasks

- [ ] **Todos los unknowns** tienen Discovery Task estructurado
- [ ] **Cada Discovery Task** tiene: owner, method, priority
- [ ] **Discovery Tasks no bloquean** el progreso (Lite puede avanzar)

## Output: Business Model Profile

- [ ] **Summary generado** (tipo, motion, funnel, unit economics, discovery tasks, implicación)
- [ ] **Sector benchmark** comparado (CAC, LTV:CAC target, payback)
- [ ] **Lite criteria met**: B2B/B2C + revenue model + acquisition sources + PLG/MLG/Sales + discovery tasks
- [ ] **Edge cases manejados** (pre-revenue, multiple models, no funnel, FinTech)

## Almacenamiento

- [ ] **Slug identificado** correctamente
- [ ] **Guardado en** `brand/{{slug}}/business-model/business-model.current.md`
- [ ] **Versionado** correcto (v1.md, history.json)
- [ ] **Link generado** para el usuario

## META (calidad)

- [ ] **Cada dato tiene fuente** (user input, competitor analysis, pricing page, inferred)
- [ ] **0 datos inventados** — todo rastreable
- [ ] **Coherencia** con company-context (elevator pitch ↔ modelo ↔ motion)
- [ ] **Unit economics coherentes** (LTV > CAC, payback razonable para el modelo)
- [ ] **Funnel template** correctamente llenado (pasos marcados measured/estimated/unknown)

---

## Flujo de uso

```
1. Agente ejecuta los 3 steps
2. Al terminar, lee este checklist
3. Marca cada ítem:
   - ✅ = completado
   - ⚠️ = no aplica o Discovery Task asignado (con razón)
   - ❌ = falta — volver a investigar
4. Si hay ❌ → completar antes de entregar
5. SOLO ENTONCES guardar y entregar al usuario
```

**No se entrega ningún documento con ❌ pendientes.**
