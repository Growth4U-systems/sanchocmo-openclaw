# Existing Customer Data — Self-QA Checklist

> Para cada ítem: ✅ completado | ⚠️ no disponible (con justificación) | ❌ pendiente
> Solo entregar con 0 ❌.

---

## Prerequisites
- [ ] **Customer count > 50** (minimum for patterns)
- [ ] **CRM access** available (export, API, or manual summary)
- [ ] **Minimum fields**: email, signup date, plan/tier, MRR
- [ ] **Skip conditions checked** — if any apply, mark pillar as skipped

## Analysis
- [ ] **RFM Segmentation** ejecutada (5 segments identified)
- [ ] **Champions profile** documentado (characteristics + why they're best)
- [ ] **Behavioral clustering** completado (3-5 clusters)
- [ ] **Churn patterns** identificados (when, why, who, triggers)
- [ ] **LTV analysis** por segmento
- [ ] **Upgrade patterns** documentados (conversion rates, triggers)

## Output
- [ ] **Overview** con métricas clave (total, MRR, churn, LTV)
- [ ] **RFM table** con acciones por segmento
- [ ] **Behavioral clusters** con características
- [ ] **Churn patterns** con early warning signals
- [ ] **Recommendations** para ICP, positioning, content, outreach

## META
- [ ] **Privacy**: 0 PII en el documento (solo aggregated data)
- [ ] **Patterns validated**: no outlier-driven conclusions
- [ ] **Actionable**: cada finding tiene recommended action
- [ ] **Coherencia** con brand files (company-context, ECPs)

## Almacenamiento
- [ ] **Slug identificado** correctamente
- [ ] **Guardado en** `brand/{{slug}}/customer-data/customer-data-current.md`
- [ ] **Versionado** correcto
- [ ] **Link generado**
