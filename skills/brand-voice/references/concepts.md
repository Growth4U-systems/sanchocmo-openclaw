# Brand Voice — Concepts & Methodology

## Two Execution Modes

| Mode | Layer | Time | When | Output |
|------|-------|------|------|--------|
| **Quick** | 0 (always-first) | ~30 min | Start of every engagement | Voice Snapshot + light Visual Notes |
| **Full** | 4 (dependent) | ~2-3 hours | After ECPs + positioning decided | Voice Guide + AI Brand Kit + Per-ECP/Channel Adaptation |

Quick mode is MANDATORY before any content creation. Full mode triggered when ECPs + positioning ready.

> **Note**: Deep visual identity (colors, typography, design system) lives in **visual-identity** skill.

---

## Quick Mode Paths

**Path A — Has URL (preferred):**
1. Scrape website: homepage, about, product pages, blog (3-5 posts)
2. Check social profiles: LinkedIn, Twitter/X, Instagram
3. Extract voice patterns + light visual notes
4. Present for validation

**Path B — No URL:**
1. Ask 5 Quick Questions
2. Analyze any brand materials (pitch deck, brand guide, content)
3. Construct voice snapshot

### 5 Quick Questions (Path B)
1. **3 adjectives** describing how brand should feel
2. **Formal ↔ casual** (1-5 scale)
3. **Technical or simple** language?
4. **Brand you admire** for communication — what specifically?
5. **Words/phrases you hate** in your industry?

---

## Full Mode: Extract vs Build

**High confidence Quick Snapshot** → Extract mode (codify existing)
**Medium** → Hybrid (extract + build)
**Low** → Build mode (construct strategically)

### Extract Mode: 6 Pattern Categories
Tone, Vocabulary, Rhythm, Structural, Personality Signals, POV — detailed in `references/prompt.md`.

### Build Mode: 15 Strategic Questions (5 blocks)
Identity (4Q), Audience (3Q), Positioning (3Q), Aspiration (2Q), Practical (3Q) — detailed in `references/prompt.md`.

## Per-ECP Tone Adaptation

Core voice stays the same. Tone ADAPTS per ECP:
- **Tone shift**: Which spectrum dimensions move
- **Vocabulary shift**: Technical terms for technical ECPs, simpler for non-technical
- **Proof emphasis**: Which trust signals matter
- **Content type priority**: Where ECP consumes content

---

## Per-Channel Guidance

Same voice, different energy:

| Channel | Typical Flex |
|---------|-------------|
| LinkedIn | More authoritative, data-backed |
| Twitter/X | Sharper, bolder, more direct |
| Email | Warmer, more personal |
| Blog | Most detailed, educational |
| Landing page | Most persuasive, benefit-driven |
| Ads | Sharpest, highest-impact |

Only fill channels the client actively uses.

---

## AI Brand Kit (KEY Deliverable)

2-3 page condensed, AI-loadable document for downstream skills:
- Voice DNA (personality, tone, POV, relationship)
- Do This / Not That (5-10 pairs with Why)
- Vocabulary Rules (USE + AVOID + jargon level)
- Rhythm (sentences, paragraphs, openings)
- Channel Quick-Reference
- Per-ECP Cheat Sheet

---

## Voice Test (5 Checks)

1. **Recognizable**: Identifiable without byline?
2. **Actionable**: Writer can produce on-brand with just AI Brand Kit?
3. **Differentiated**: Sounds different from competitors?
4. **Authentic**: Feels true to who they are?
5. **Consistent**: Works across formats?

If any fails: identify which, generate 3 samples, review with client, adjust, re-check.

---

## Lite vs Deep

**Lite** (Quick complete): Voice Snapshot + Visual Notes + words USE/AVOID (5+) + Do/Don't (3+ pairs) + examples by type
**Deep** (Full complete): All Lite + Voice Profile + AI Brand Kit + Per-ECP + Per-Channel + Do/Don't (5-10) + Voice Test passed + 1+ iteration

---

## Edge Cases

- **Inconsistent across channels**: Extract all, ask client to unify or maintain intentional variation
- **Multiple founders**: Define company voice + note how each founder relates
- **B2B with different ECPs**: Same core voice, Per-ECP adaptation handles shift
- **No existing content**: Build mode mandatory, use competitor voice as reference
- **Client wants to sound like X**: Analyze X, extract what to emulate, ADAPT (never copy)
- **Multi-language**: Core personality universal, vocabulary/examples per-language natively
