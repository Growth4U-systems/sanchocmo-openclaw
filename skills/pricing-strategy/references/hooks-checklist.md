# Pricing Hooks — Checklist de QA

## Datos y Fuentes
- [ ] Todos los precios de competidores tienen URL de fuente
- [ ] No hay precios inventados o estimados sin marcar
- [ ] Datos del cliente verificados contra docs (business-model, budget)
- [ ] 🔴 DUDA marcado donde falta info (no rellenado con suposiciones)

## Análisis Competitivo
- [ ] Mínimo 3 competidores con precios documentados
- [ ] Tabla comparativa con columnas: competidor, servicio, precio, modelo, fuente
- [ ] Rango de mercado identificado (min-max-media)
- [ ] Modelo de pricing de cada competidor documentado

## Estructura de Precios
- [ ] Propuesta coherente con el sector del cliente
- [ ] Precios dentro del rango de mercado (o justificación si no)
- [ ] Cada tier tiene: nombre, servicios, precio, target, justificación
- [ ] Considerado pricing público vs privado

## Hooks Psicológicos
- [ ] Mínimo 5 hooks aplicados
- [ ] Cada hook tiene ejemplo CONCRETO (no genérico)
- [ ] Hooks relevantes al sector (no todos aplican a todos)
- [ ] Especificado DÓNDE aplicar cada hook

## Coherencia Cross-Pillar
- [ ] Pricing coherente con positioning (premium = precio premium)
- [ ] Pricing coherente con business-model (márgenes sostenibles)
- [ ] Pricing coherente con budget (objetivos de revenue)
- [ ] Pricing coherente con SWOT (aprovecha fortalezas, mitiga debilidades)

## Plan de Implementación
- [ ] Quick wins identificados (1-2 semanas)
- [ ] Métricas definidas
- [ ] A/B tests propuestos
- [ ] Cronograma realista

## Formato
- [ ] Versión tag en header (`v1.0`)
- [ ] Self-QA metadata al final
- [ ] Estructura clara con headers
- [ ] Sin bloques de texto sin formatear
