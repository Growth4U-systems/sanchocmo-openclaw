# Platforms Guide - Where to Steal Ideas

## Facebook Ads Library

**URL**: `https://www.facebook.com/ads/library/`

**Search by:**
- Competitor company name
- Competitor brand variations
- Product name

**Filters:**
- ✅ Active ads only (currently running)
- ✅ All countries or specific geo
- ✅ Media type (image, video, carousel)

**What you can see:**
- Ad creative (images/videos)
- Ad copy (headline + body text)
- CTA button text
- When ad started running
- Platforms (FB, IG, Messenger, Audience Network)
- Ad variations (A/B tests)

**What you CANNOT see:**
- Targeting (age, interests, behaviors)
- Budget or spend
- Performance metrics (CTR, conversions)

**Red flags:**
- If competitor has 0 active ads → They stopped paid ads (why?)
- If competitor has 50+ variants → Heavy testing (winning formula unclear)

## LinkedIn (Profile & Company Page)

**URLs:**
- Company: `https://linkedin.com/company/{company-slug}`
- Personal: `https://linkedin.com/in/{person-slug}`

**What to extract:**
- **Last 30 posts** - Content calendar analysis
- **Post format** - Text, image, carousel, video, poll
- **Engagement** - Reactions, comments, reposts
- **Posting frequency** - Cadence and timing
- **Employee amplification** - Do employees reshare company content?

**Tools:**
- Stealth browser (avoid detection)
- Scroll to load ~60 posts
- Extract HTML for structured data

## Twitter/X

**URL**: `https://twitter.com/{handle}`

**What to extract:**
- **Tweet frequency** - Tweets per day
- **Thread usage** - Long-form storytelling
- **Reply strategy** - Engagement in comments
- **Retweet patterns** - What they amplify
- **Media usage** - Images, videos, GIFs

**API access:**
- If available, use official API
- Else: Browser scraping

## Product Hunt

**URL**: `https://producthunt.com/@{company}`

**What to look for:**
- **Launch posts** - How they positioned the product
- **Tagline** - One-liner messaging
- **Maker comment** - Founder's personal pitch
- **Community response** - What resonated in comments
- **Upvotes** - Validation signal

## Changelog Pages

**Common URL patterns:**
- `{domain}/changelog`
- `{domain}/updates`
- `{domain}/whats-new`
- `{domain}/releases`
- `{domain}/blog/category/product-updates`

**What to extract:**
- **Feature releases** - New capabilities launched
- **Frequency** - How often they ship
- **Messaging** - How they announce features
- **User feedback** - Comments, requests mentioned

**Competitors to watch:**
- Direct competitors (from Battle Cards)
- Adjacent tools (different solution, same problem)
- Aspirational products (where the market is going)

## Landing Pages

**URLs:**
- Homepage: `{domain}`
- Product pages: `{domain}/product` or `{domain}/features`
- Pricing: `{domain}/pricing`
- Use Cases: `{domain}/use-cases/*`

**What to extract:**

### Hero Section
- **Headline** - Main value prop
- **Subheadline** - Supporting explanation
- **CTA** - Primary button text
- **Visual** - Product screenshot, illustration, video

### Copy Patterns
- **Pain → Solution** - How they frame the problem
- **Benefits vs Features** - Outcome-focused or spec-focused
- **Social Proof** - Logos, testimonials, numbers
- **Objection handling** - What fears they address

### Structure
- **Sections order** - Hero → Problem → Solution → Features → Social Proof → Pricing → CTA
- **Visual hierarchy** - Bold claims, section spacing
- **Conversion points** - How many CTAs, where placed

## Review Sites (Bonus Intel)

**G2**: `https://g2.com/products/{product}/reviews`
**Capterra**: `https://capterra.com/software/{id}/reviews`
**Trustpilot**: `https://trustpilot.com/review/{domain}`

**What to extract:**
- **Pros** (mentioned by users) - Competitive strengths
- **Cons** (mentioned by users) - Opportunities to differentiate
- **Alternatives considered** - Who they compete with
- **Use cases** - Who uses it and for what

**Idea generation:**
- Cons = Opportunities for our messaging ("Unlike X, we...")
- Pros = Table stakes (we need these too)
- Alternatives = Expand competitive set
