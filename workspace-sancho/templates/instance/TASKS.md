# TASKS — Client Tasks

> Client-specific tasks. System/infrastructure tasks go in Cervantes TASKS.md.

## How it works

1. **Propose**: From client Discord or direct to agent
2. **Agent registers**: Task here with status `proposed`
3. **Review**: Admin approves/rejects
4. **Execute**: Only `approved` tasks get executed
5. **Close**: On completion -> `done`

## Statuses

| Emoji | Status | Who |
|---|---|---|
| inbox | Proposed | Anyone proposes |
| eyes | In review | Agent analyzes impact |
| check | Approved | Admin approves |
| x | Rejected | Admin rejects (with reason) |
| wrench | In progress | Agent executes |
| done | Done | Completed |
| ice | Frozen | Parked |
