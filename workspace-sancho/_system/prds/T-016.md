# PRD — T-016: Auto-binding Discord channels al crear cliente

## Objetivo

Script que lee los canales del guild via Discord API, matchea por nombre con patrones conocidos, genera bindings y actualiza `openclaw.json` automáticamente al crear un nuevo cliente.

## Contexto y motivación

- Cada cliente tiene un guild de Discord con canales (#campaigns, #intelligence, #content, etc.).
- Actualmente los bindings (agente → canal) se configuran manualmente en openclaw.json.
- Con la estructura de 4 agentes (Sancho default + Rocinante/Escudero internos), los bindings son más simples, pero aún hay que mapear Cervantes a #admin.
- Al escalar a múltiples clientes, configurar manualmente es error-prone.

## Requisitos funcionales

- **RF-1**: Script que recibe guild ID como input
- **RF-2**: Lee canales del guild via Discord API (bot token existente)
- **RF-3**: Matchea canales por nombre a patrones: `admin` → cervantes, resto → sancho (default)
- **RF-4**: Genera array de bindings compatible con openclaw.json
- **RF-5**: Opción `--dry-run` para preview sin escribir
- **RF-6**: Opción `--apply` para escribir directamente en openclaw.json
- **RF-7**: Idempotente — no duplica bindings existentes

## Archivos afectados

| Archivo | Acción |
|---------|--------|
| `scripts/auto-bind.sh` o `scripts/auto-bind.py` | Crear |
| `openclaw.json` | Modificar (solo con --apply) |

## Pasos de implementación

1. Script Python/bash que usa Discord API (`/guilds/{id}/channels`)
2. Pattern matching configurable (map de nombre → agente)
3. Genera bindings JSON
4. Merge con bindings existentes (sin duplicar)
5. Dry-run output + apply mode
6. Test con guild Hospital Capilar

## Criterios de aceptación

- [ ] Script lee canales de un guild dado
- [ ] Genera bindings correctos (cervantes → admin, sancho → default)
- [ ] `--dry-run` muestra preview sin modificar config
- [ ] `--apply` escribe en openclaw.json correctamente
- [ ] No duplica bindings existentes
- [ ] Documentado en TOOLS.md

## Riesgos y dependencias

| Riesgo | Mitigación |
|--------|-----------|
| Discord API rate limits | Una sola llamada por ejecución; no es problema |
| Nombres de canales no estándar | Pattern matching configurable, fallback a sancho |
| openclaw.json corrupto si falla mid-write | Escribir a .tmp primero, luego rename atómico |
