# PRD — T-031: Context Matrix enforcement en skills

## Objetivo

Añadir un bloque `CONTEXT_REQUIRED` explícito a cada skill que liste SOLO los archivos que debe leer. Actualizar SOUL.md para que Sancho pase solo esos archivos al spawnear Escuderos.

## Contexto y motivación

- `_system/brand-memory.md` define qué archivos pasa cada skill (Context Matrix). Pero nada impide que Sancho o un Escudero cargue todo `./brand/`.
- Sin enforcement, cada skill lee todo el Context Lake → tokens desperdiciados, contexto diluido, riesgo de contaminar output con info irrelevante.
- Un Escudero que recibe todo `brand/` para escribir un email subject line está quemando tokens innecesariamente.
- Cada skill debe declarar exactamente qué necesita — ni más ni menos.

## Requisitos funcionales

- **RF-1**: Bloque `CONTEXT_REQUIRED` en frontmatter de cada SKILL.md:
  ```yaml
  context_required:
    - brand/company-context.md
    - brand/positioning.md
    - brand/voice-profile.md
  ```
- **RF-2**: Bloque `CONTEXT_WRITES` para declarar dónde escribe:
  ```yaml
  context_writes:
    - brand/company-context.md    # (solo skills de Foundation)
    - campaigns/{slug}/brief.md   # (skills de ejecución)
  ```
- **RF-3**: Regla en SOUL.md de Sancho:
  ```
  Cuando spawnes Escudero con sessions_spawn:
  - Lee CONTEXT_REQUIRED del skill que va a ejecutar
  - Pasa SOLO esos archivos como contexto
  - NUNCA pases todo ./brand/
  ```
- **RF-4**: Actualizar las 44 skills con CONTEXT_REQUIRED y CONTEXT_WRITES correctos
- **RF-5**: Script de validación: `scripts/validate-context-matrix.py` que verifica:
  - Todas las skills tienen CONTEXT_REQUIRED
  - No hay archivos referenciados que no existen
  - CONTEXT_WRITES respeta la regla brand/ vs campaigns/

## Pasos de implementación

1. Auditar cada skill: qué archivos lee realmente y qué escribe
2. Definir CONTEXT_REQUIRED y CONTEXT_WRITES por skill
3. Script batch para añadir los bloques al frontmatter
4. Actualizar SOUL.md con regla de enforcement
5. Crear script de validación
6. Test: verificar que Sancho respeta el context al spawnear Escudero

## Criterios de aceptación

- [ ] 44/44 skills tienen CONTEXT_REQUIRED
- [ ] 44/44 skills tienen CONTEXT_WRITES
- [ ] SOUL.md tiene regla explícita de Context Matrix
- [ ] Script de validación pasa sin errores
- [ ] Escuderos reciben solo archivos necesarios (verificar en logs)
- [ ] brand-memory.md alineado con frontmatter de skills

## Riesgos y dependencias

| Riesgo | Mitigación |
|--------|-----------|
| 44 skills es mucho trabajo manual | Script batch; la mayoría comparten patrones (Foundation skills leen company-context, execution skills leen positioning+voice) |
| OpenClaw no parsea context_required nativamente | Es una convención; el enforcement es en SOUL.md (instrucción al modelo), no técnico |
| Skills nuevas olvidan añadir el bloque | Script de validación en CI / heartbeat check |
