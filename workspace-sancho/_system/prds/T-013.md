# PRD — T-013: Estilo visual SanchoCMO en Mission Control

## Objetivo

Aplicar una identidad visual coherente al dashboard Mission Control: paleta de colores SanchoCMO, tipografía, logo/icono, favicon, header branding e ilustraciones generadas con nano-banana-pro. Transformar el dashboard actual (genérico, oscuro/claro sin personalidad) en algo que se sienta como producto propio de SanchoCMO.

## Contexto y motivación

- Mission Control es el dashboard HTML estático servido via Tailscale (`/mc`), regenerado por `regenerate.py`.
- Actualmente usa estilos genéricos: colores planos (`--purple: #a78bfa`, etc.), tipografía del sistema (`-apple-system`), sin logo, sin favicon, sin ilustraciones.
- Existe un **Comic UI Design System** (`skills/comic-ui-system/SKILL.md`) que define la identidad visual de SanchoCMO: halftone, bordes gruesos, tipografía Space Grotesk/Nunito, colores cálidos (parchment, rust, ink).
- El dashboard debería sentirse como SanchoCMO, no como un template genérico.
- Las ilustraciones (mascota Sancho, iconos de sección, empty states) se generarán con **nano-banana-pro** (Gemini 3 Pro Image).

## Requisitos funcionales

### RF-1: Paleta de colores
- Reemplazar las CSS variables actuales (`--bg`, `--card`, `--border`, `--text`, `--purple`, etc.) por la paleta SanchoCMO del Comic UI System:
  - Fondos: parchment (#F5E6D0), paper (#FAF0E6), aged (#EDE0D0)
  - Bordes: ink (#1A1A2E), UI border (#2D2D44)
  - Acentos: rust (#C84B31), warm-gold (#DAA520), olive (#556B2F)
  - Texto: ink (#1A1A2E), muted (#5D5348)
- Mantener dark mode como opción (invertir parchment → dark ink)

### RF-2: Tipografía
- **H1-H2**: Space Grotesk (display headlines)
- **H3-H6**: Nunito Bold
- **Body/UI**: Nunito Regular
- **Narrator/quotes**: Playfair Display Italic
- Cargar via Google Fonts (`<link>` en `<head>`)

### RF-3: Logo y favicon
- Generar logo SanchoCMO con nano-banana-pro: Sancho Panza estilizado como CMO (traje, gráficas, lanza = puntero)
- Formato: SVG o PNG 512x512
- Favicon: versión reducida 32x32 + 180x180 (apple-touch-icon)
- Reemplazar el texto "🐴 SanchoCMO" en nav por logo + texto

### RF-4: Header branding
- Nav lateral: logo arriba, versión badge con estilo Comic UI (borde grueso, sombra offset)
- Header de cada página: título con Space Grotesk, subtítulo con Nunito
- Secciones separadas con bordes ink de 4px (estilo panel de cómic)

### RF-5: Ilustraciones
- Generar con nano-banana-pro (mínimo 4):
  1. **Mascota Sancho** — para el header/nav (estilo cómic, cálido)
  2. **Empty state** — para cuando no hay datos ("Sancho buscando datos...")
  3. **Icono Foundation** — para la sección de pillars
  4. **Icono Tasks** — para el kanban de tareas
- Estilo: línea gruesa, colores cálidos, halftone, vintage print
- Guardar en `assets/mc/` como PNG

### RF-6: Estilos de componentes
- Cards: bordes 4px ink, sombra offset 6px (flat, sin blur)
- Botones: estilo Comic UI (bordes gruesos, hover con rotación sutil)
- Badges/pills: bordes redondeados, colores sólidos SanchoCMO
- Stats: números en Space Grotesk, labels en Nunito
- Phase bar: estilo panel de cómic con halftone en fondo

### RF-7: Halftone y texturas
- Fondo general: textura papel vía SVG `feTurbulence` (no imágenes)
- Halftone dots en áreas de acento: `radial-gradient`
- Paper grain sutil en cards

## Pasos de implementación

1. **Generar assets** — Usar nano-banana-pro para logo, favicon, mascota e ilustraciones. Iterar hasta que el estilo sea coherente. Guardar en `assets/mc/`.

2. **Actualizar CSS variables** — Reemplazar la paleta actual por la paleta SanchoCMO. Ajustar dark mode.

3. **Cargar tipografía** — Añadir Google Fonts (Space Grotesk, Nunito, Playfair Display) en `<head>`. Aplicar a selectores.

4. **Aplicar bordes y sombras Comic UI** — Cards, nav, stat boxes, phase bar. Seguir las reglas del design system (4px borders, 6px offset shadows, flat).

5. **Añadir texturas** — Halftone y paper grain via CSS (feTurbulence, radial-gradient). Sin imágenes externas.

6. **Integrar logo y favicon** — Reemplazar emoji 🐴 por logo. Añadir `<link rel="icon">` y apple-touch-icon.

7. **Integrar ilustraciones** — Empty states, iconos de sección, mascota en nav.

8. **Actualizar regenerate.py** — Asegurar que los nuevos assets se sirven correctamente y que el path es relativo.

9. **Testing** — Verificar en light/dark mode. Verificar en Tailscale serve. Verificar responsive.

## Criterios de aceptación

- [ ] Dashboard usa paleta SanchoCMO (no colores genéricos)
- [ ] Tipografía Space Grotesk + Nunito + Playfair cargada y aplicada
- [ ] Logo y favicon visibles en nav y pestaña del navegador
- [ ] Al menos 4 ilustraciones generadas e integradas
- [ ] Bordes 4px y sombras offset 6px en cards (estilo Comic UI)
- [ ] Textura papel y halftone visibles en fondo
- [ ] Dark mode funcional con paleta invertida
- [ ] `regenerate.py` ejecuta sin errores
- [ ] Dashboard accesible via Tailscale serve en `/mc`

## Riesgos y dependencias

| Riesgo | Mitigación |
|--------|-----------|
| nano-banana-pro genera imágenes inconsistentes en estilo | Iterar con prompts específicos; definir estilo de referencia en el primer asset y reusar como seed |
| Tipografía Google Fonts no carga offline | Incluir fallback a system fonts en font-stack |
| Peso de imágenes ralentiza carga | Comprimir PNGs; usar lazy loading para ilustraciones |
| Comic UI system es complejo, fácil romper reglas | Seguir las Hard Boundaries del SKILL.md al pie de la letra |
| T-010 (migración a Next.js) haría obsoleto parte de este trabajo | Este PRD se aplica al HTML actual; el estilo CSS/assets se reutilizan en Next.js |
