# PRD — T-035: URLs obligatorias en market-intelligence + sugerencia deep-research

## Objetivo

Que todo output de market-intelligence incluya URLs de fuente verificables para cada cifra/dato, y que al finalizar sugiera al usuario profundizar con deep-research.

## Contexto y motivación

Market-intelligence genera datos de mercado (TAM, SAM, SOM, tendencias, competidores, cifras de sector) pero actualmente no cita fuentes. Esto genera:
- **Desconfianza**: El usuario no puede verificar si los datos son reales o alucinados
- **Inutilidad profesional**: En presentaciones a clientes, cifras sin fuente no sirven
- **Falta de profundidad**: El skill hace un barrido rápido pero no ofrece opción de ir más profundo

La sugerencia de deep-research al final cierra el loop: "esto es lo rápido, ¿quieres lo profundo?"

## Requisitos funcionales

### RF-1: URLs obligatorias en output
- Toda cifra, dato estadístico o afirmación cuantitativa debe incluir `[Fuente: título](url)` inline
- Si no se encuentra fuente verificable para un dato → marcarlo explícitamente como `⚠️ Estimación sin fuente verificada`
- Mínimo 5 fuentes por sección de mercado
- Fuentes aceptables: informes de consultoras (Statista, IBISWorld, Grand View Research), prensa especializada, publicaciones oficiales (INE, Eurostat), SEC filings, earnings calls
- NO aceptar: blogs genéricos, artículos sin fecha, fuentes anteriores a 2023

### RF-2: Validación de URLs
- Cada URL citada debe haberse visitado/verificado con `web_fetch` o `web_search` durante la ejecución
- No inventar URLs — solo citar lo que se ha encontrado realmente
- Si una búsqueda no devuelve resultados fiables → documentarlo como gap de información

### RF-3: Sugerencia de deep-research al final
- Al completar el output, añadir bloque:
  ```
  ---
  📊 **¿Quieres profundizar?**
  Puedo lanzar deep-research para ampliar este análisis con más fuentes, datos más granulares y validación cruzada.
  → Escribe "profundizar" o "deep-research" para continuar.
  ```
- Si el usuario acepta → invocar deep-research con el doc generado como input base

### RF-4: Formato de citación en el documento final
- Sección `## Fuentes` al final del documento con lista numerada de todas las URLs citadas
- Cada fuente: `[N] Título — Organización (fecha). URL`
- Referencias inline: `dato relevante [N]`

## Pasos de implementación

1. **Editar SKILL.md** — Añadir sección "Reglas de citación" con RF-1 a RF-4
2. **Crear `references/citation-guidelines.md`** — Fuentes aceptables, formato, ejemplos
3. **Actualizar templates de output** — Añadir sección `## Fuentes` y placeholder de deep-research
4. **Añadir instrucción de web_search/web_fetch** — El skill debe buscar activamente fuentes para cada dato
5. **Añadir bloque de sugerencia deep-research** al final del flow
6. **Test**: Ejecutar market-intelligence para Hospital Capilar y verificar que todas las cifras tienen URL

## Criterios de aceptación

- [ ] Toda cifra/dato en el output tiene `[Fuente](url)` o `⚠️ Estimación sin fuente`
- [ ] Sección `## Fuentes` al final con lista numerada
- [ ] Mínimo 5 fuentes verificables por sección
- [ ] URLs realmente visitadas durante ejecución (no inventadas)
- [ ] Bloque "¿Quieres profundizar?" aparece al final
- [ ] Si usuario dice "profundizar" → se invoca deep-research correctamente

## Riesgos y dependencias

| Riesgo | Mitigación |
|--------|-----------|
| web_search no devuelve fuentes para nichos muy específicos | Marcar como gap, sugerir deep-research para búsqueda más exhaustiva |
| Aumento de tokens por búsquedas de fuentes | Aceptable — la calidad lo justifica. Estimado +30% tokens por ejecución |
| deep-research no existe aún (T-036) | La sugerencia se puede añadir ya; la invocación se activa cuando T-036 esté implementado |
| Fuentes pueden quedar obsoletas | Incluir fecha de acceso en cada fuente |

## Estimación de esfuerzo

- **Edición de SKILL.md + templates**: 30 min
- **Crear citation-guidelines.md**: 15 min
- **Testing con Hospital Capilar**: 30 min
- **Total**: ~1.5h (puede hacerlo Cervantes directamente)
- **Dependencia**: T-036 para la parte de invocación de deep-research
