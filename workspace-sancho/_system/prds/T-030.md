# PRD — T-030: Skill funnel-architect

## Objetivo

Crear skill para diseñar funnels de cualificación y conversión: quiz funnels, webinar funnels, free trial funnels, consultation funnels. Desde la arquitectura hasta los assets necesarios.

## Contexto y motivación

- Sancho improvisó un quiz funnel para Hospital Capilar sin skill dedicada. El resultado fue decente pero no seguía un framework probado.
- Las skills actuales cubren piezas individuales (lead-magnet, email-sequences, direct-response-copy) pero ninguna diseña la **arquitectura completa del funnel**: qué pasos, qué cualifica, qué descualifica, dónde está el handoff a ventas.
- Un funnel mal diseñado desperdicia todo el trabajo de las skills downstream.

## Requisitos funcionales

- **RF-1**: Tipos de funnel soportados:
  - Quiz funnel (cualificación por respuestas)
  - Webinar funnel (educación → oferta)
  - Free trial / freemium funnel
  - Consultation funnel (agendar cita)
  - Product launch funnel
  - VSL (Video Sales Letter) funnel
- **RF-2**: Para cada funnel, definir:
  - Etapas (awareness → consideration → decision → action)
  - Criterios de cualificación por etapa
  - Puntos de descualificación (no todos son clientes)
  - Handoff points (dónde entra el equipo humano)
  - Métricas por etapa (conversion rates esperados)
  - Assets necesarios por etapa (landing, email, ad, content)
- **RF-3**: Lee de Foundation: positioning, ECPs, budget, channel-plan
- **RF-4**: Output: `campaigns/<slug>/funnel-architecture.md` con:
  - Diagrama del funnel (markdown/mermaid)
  - Brief por etapa
  - Lista de assets necesarios (input para otras skills)
  - Métricas target
- **RF-5**: Integración con gtm-orchestrator (T-029): funnel-architect es paso 4 del DAG

## Pasos de implementación

1. Investigar frameworks de funnel design (Russell Brunson, Ryan Deiss, ClickFunnels)
2. Definir templates por tipo de funnel
3. Escribir SKILL.md con instrucciones, frameworks, y outputs esperados
4. Integrar lectura de Foundation (positioning, ECPs, channel-plan)
5. Definir formato de output (funnel-architecture.md)
6. Test con Hospital Capilar (quiz funnel para tratamientos capilares)
7. Añadir a dispatch de gtm-orchestrator

## Criterios de aceptación

- [ ] Skill diseña funnel completo con etapas, criterios, y assets
- [ ] Soporta al menos 3 tipos de funnel
- [ ] Lee Foundation como contexto
- [ ] Output en campaigns/ (no en brand/)
- [ ] Incluye métricas target por etapa
- [ ] Lista de assets necesarios para downstream skills
- [ ] Integrado en gtm-orchestrator DAG

## Riesgos y dependencias

| Riesgo | Mitigación |
|--------|-----------|
| Demasiados tipos de funnel → skill enorme | Empezar con quiz + consultation (los más comunes para servicios); añadir tipos después |
| Depende de Foundation completa | Hard check: si positioning vacío → error |
| Output demasiado teórico sin accionable | Cada etapa debe tener brief concreto que otra skill puede ejecutar |
