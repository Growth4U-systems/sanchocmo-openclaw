# PRD — T-017: Script new-client.sh

## Objetivo

Script que crea el workspace completo para un nuevo cliente: directorio, subdirectorios, archivos template, sources.json vacío. Input: slug del cliente. Output: workspace listo para arrancar Foundation.

## Contexto y motivación

- Cada cliente necesita una estructura de workspace idéntica para que Sancho y las skills funcionen.
- Crearlo manualmente es tedioso y error-prone (olvidar un subdirectorio, un archivo, etc.).
- Con un script estandarizado, onboardear un cliente nuevo es un solo comando.

## Requisitos funcionales

- **RF-1**: Input: `new-client.sh <slug>` (ej: `new-client.sh hospital-capilar`)
- **RF-2**: Crea directorio `~/.openclaw/workspace-sancho/clients/<slug>/`
- **RF-3**: Crea subdirectorios: `brand/`, `campaigns/`, `intelligence/`, `assets/`, `reports/`
- **RF-4**: Crea archivos template:
  - `brand/company-context.md` (template vacío con headers)
  - `brand/positioning.md` (template)
  - `brand/competitors.md` (template)
  - `brand/voice-profile.md` (template)
  - `sources.json` (vacío: `{}`)
  - `.env.template` (variables necesarias por cliente)
  - `README.md` (nombre del cliente, fecha creación, estado)
- **RF-5**: No sobreescribe si el directorio ya existe (safety check)
- **RF-6**: Output: resumen de lo creado + siguiente paso ("Ejecuta /sancho_start en Discord")

## Archivos afectados

| Archivo | Acción |
|---------|--------|
| `scripts/new-client.sh` | Crear |
| `scripts/templates/` | Crear — templates de archivos por defecto |

## Pasos de implementación

1. Crear script bash con validación de input
2. Definir estructura de directorios estándar
3. Crear templates para cada archivo base
4. Safety check (directorio existente)
5. Output con resumen y next steps
6. Test: crear cliente dummy, verificar estructura

## Criterios de aceptación

- [ ] `new-client.sh hospital-capilar` crea estructura completa
- [ ] Todos los subdirectorios y archivos template presentes
- [ ] No sobreescribe cliente existente
- [ ] Templates tienen headers correctos (skill-compatible)
- [ ] README.md incluye nombre, fecha, estado inicial
- [ ] Script documentado con `--help`

## Riesgos y dependencias

| Riesgo | Mitigación |
|--------|-----------|
| Estructura de cliente cambia con el tiempo | Templates versionados; script lee templates desde directorio, no hardcoded |
| Multi-client routing (T-021) necesita registro central | Script puede añadir entrada a clients.js como paso opcional |

## Estimación de esfuerzo

| Tarea | Tiempo |
|-------|--------|
| Script + templates | 1.5h |
| Safety checks + output | 30min |
| Test + docs | 30min |
| **Total** | **~2.5h** |
