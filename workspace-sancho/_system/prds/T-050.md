## Problema

Growth4U gestiona leads en GoHighLevel (GHL) pero:
1. Las notas de seguimiento viven en threads de Slack y no llegan a GHL
2. No hay inteligencia contextual sobre los leads (empresa, transcripts de llamadas)
3. No hay briefings automatizados antes de las llamadas
4. El equipo comercial entra a calls sin contexto → menor tasa de cierre

## Objetivo

Sistema automático que:
- **Sincroniza** notas de Slack → GHL (fuente de verdad operativa)
- **Enriquece** leads nuevos con company research + contact data + transcripts
- **Prepara** briefings de llamadas (diario + semanal)
- **Publica** resultados en Discord #intelligence

## Mejoras v2 vs rama original

| Área | Rama original | v2 mejorada |
|------|--------------|-------------|
| **Delegación** | Crons llaman a Sancho directamente | Sancho orquesta, Escudero ejecuta (dispatch-protocol) |
| **Modelo** | Sonnet para todo | Sonnet para enrichment, Opus solo para inferencia de objetivo en call prep |
| **Versionado** | Archivos sueltos | `leads/{ghl_id}/current.md` + `history.json` (versioning-protocol) |
| **Multi-cliente** | Hardcoded Growth4U | Config por slug, env `{SLUG_UPPER}_GHL_API_KEY`, script `for-each-client.sh` ready |
| **Active flag** | `active: true` (rompe otros crons) | Nuevo campo `enabledFeatures: ["lead-tracking"]` |
| **Filtro de enrichment** | Todos los contactos GHL | Solo leads con tag `llamada-agendada` (reduce de ~200 a ~10-20) |
| **Foundation gate** | No existe | Solo enrich si Foundation existe (company-brief mínimo) |
| **HEARTBEAT** | Duplica crons | Solo verifica que crons corrieron OK |
| **Error recovery** | Básico | 3-attempt retry (skill-communication-protocol) |
| **Client isolation** | No mencionado | Briefings solo en guild Growth4U, nunca cross-client |
| **Cron IDs** | Formato libre | UUIDs válidos |
| **MC links** | No hay | Links tokenizados a lead files en MC |
| **Dry-run mode** | No existe | Flag `dry_run` en config para testear sin escribir |
| **Health check** | No existe | Ping GHL API al inicio, abort + alerta si falla |
| **Error alerting** | Básico | Alerta explícita si Slack sync disabled, max 3 errores/hora |

## Componentes

### 1. Skill: `lead-intelligence-hub`

**Frontmatter:**
```yaml
name: lead-intelligence-hub
description: "Syncs leads from GHL, enriches new leads with company research and contact data, updates existing leads with latest GHL state and call transcripts. Triggers: 'sync leads', 'lead intelligence', 'enrich leads'."
metadata:
  author: Martin (Growth4U)
  version: '2.0'
  system: SanchoCMO
  phase: Encuentra
  pillar: lead-tracking
  layer: 0+4
  depends_on: [company-finder, contact-enrichment]
  chains_to: [sales-call-prep]
context_required:
  - brand/{slug}/lead-tracking-config.json
context_writes:
  - brand/{slug}/leads/
```

**Ejecución delegada:** Sancho orquesta via `sessions_spawn` (Escudero, Sonnet).

**Flujo:**
```
0. HEALTH CHECK: Ping GHL API → si falla, abort + alertar en Discord → exit
1. Lee brand/{slug}/lead-tracking-config.json
   - Si dry_run: true → ejecutar todo pero NO escribir a GHL ni crear archivos (solo log)
2. Slack notes → GHL custom fields (solo si Slack MCP disponible)
   - Si Slack MCP no disponible → publicar alerta: "⚠️ Slack sync DESACTIVADO — notas NO se sincronizan"
3. Fetch GHL contacts FILTRADOS por tag "llamada-agendada"
   - Query: GET /contacts/?locationId={id}&query=llamada-agendada (o filtrar por tag post-fetch)
   - Solo procesar contactos con este tag (típicamente 10-20, no cientos)
4. Para cada contacto filtrado SIN leads/{ghl_id}/current.md:
   a. company-finder (modo targeted, NO ICP)
      - Si falla → crear lead file con enrichment_pending: true, continuar
   b. contact-enrichment (gap-filling: solo datos que faltan)
      - Si falla → log warning, continuar con lo que hay
   c. Drive transcript search (gog drive search)
      - Límite: solo últimos 90 días en primera ejecución, 3 transcripts max por lead
      - Si falla → crear lead file sin transcripts, log warning
   d. Crear leads/{ghl_id}/current.md
5. Para cada contacto filtrado CON leads/{ghl_id}/current.md:
   a. Buscar nuevos transcripts (date > last_enriched, max 3)
   b. Si hay nuevos → actualizar current.md
   c. Si enrichment_pending: true → reintentar company-finder/contact-enrichment
6. Publicar resumen en Discord #intelligence (max 3 mensajes error/ejecución)
7. Actualizar memory/heartbeat-state.json → lead_sync
```

**Lead file template** (SOLO inteligencia, NO datos operativos de GHL):
```markdown
---
ghl_id: {id}
version: 1
last_enriched: {ISO}
enrichment_pending: false
---
<!-- version: 1 | fecha: YYYY-MM-DD | skill: lead-intelligence-hub | qa: PASS -->

## Company Intelligence
{company-finder output — sector, tamaño, producto, competidores, funding}

## Contact Enrichment
{datos extra NO en GHL: LinkedIn verificado, email secundario}

## Call Transcripts
### {date} — {call_title}
- **Pain points**: {extracted}
- **Next steps**: {extracted}
- **Key quotes**: {extracted}
- 🔗 [Transcript en Drive]({drive_link})
```

**Estructura de archivos por lead:**
```
brand/{slug}/leads/{ghl_id}/
  current.md     ← activo
  v1.md          ← histórico (si se actualiza)
  history.json   ← log de versiones
```

### 4. clients.json — Feature flags (NO `active: true`)


## Alineación con protocolos SanchoCMO

| Protocolo | Cómo se aplica |
|-----------|---------------|
| **dispatch-protocol** | lead-intel → Escudero (spawn), call-prep → Sancho directo |
| **versioning-protocol** | leads/{id}/current.md + history.json |
| **skill-communication-protocol** | 3-attempt retry con fallback model |
| **client-context-isolation** | Briefings SOLO en guild Growth4U, lead files en brand/growth4u/ |
| **mc-links-protocol** | Links a lead files via MC admin token (guild interno) |
| **foundation-protocol** | Gate check: si no hay Foundation, enrichment usa company-finder sin ECPs |
| **new-client-protocol** | Futuro: `new-client.sh` crea lead-tracking-config.json si feature habilitada |
