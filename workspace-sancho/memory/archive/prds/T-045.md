# T-045 — Fix Broken Skill Links in Mission Control

**Prioridad:** P0 (UX roto)
**Categoría:** skill
**Propuesto por:** Alfonso — 2026-03-08

**Archivos afectados:** `mission-control.html`

## Problema

La página **Skills** de Mission Control tiene **11 links rotos** que llevan a "Not Found" al hacer click. Hay **2 causas raíz independientes**:

### Causa 1: DAG usa nombres abreviados que no coinciden con los directorios reales (9 links)

La función `openSkill(name)` abre `/mc/docs/skills/{name}/SKILL.md`. Los nodos del DAG (`DAG_LAYERS`) y las dependencias (`SKILL_DEPS`) usan nombres cortos que no existen como carpetas en `skills/`:

| DAG node (roto) | Directorio real |
|---|---|
| `business-model` | `business-model-audit` |
| `budget` | `budget-constraints` |
| `market-analysis` | `market-intelligence` |
| `competitor-analysis` | `competitor-intelligence` |
| `self-analysis` | `self-intelligence` |
| `swot` | `swot-analysis` |
| `niche-discovery` | `niche-discovery-100x` |
| `positioning` | `positioning-messaging` |
| `pricing` | `pricing-strategy` |

### Causa 2: SKILL_CATS referencia skills inexistentes (2 links)

En la categoría `utils`:
| SKILL_CATS entry (roto) | Situación |
|---|---|
| `competitor-alternatives` | No existe como skill (ni carpeta ni en skills-data.js) |
| `nano-banana-pro` | Existe como skill de sistema (`/opt/homebrew/lib/...`) pero NO en el workspace `skills/` |

---

## Solución

### Paso 1: Corregir `DAG_LAYERS` — usar nombres reales de directorio

```javascript
const DAG_LAYERS = [
  {label:"📋 Company Brief (L0)",nodes:["company-context","business-model-audit","budget-constraints"]},
  {label:"📊 Research (L1)",nodes:["market-intelligence","competitor-intelligence","self-intelligence"]},
  {label:"🔄 Synthesis (L2)",nodes:["swot-analysis"]},
  {label:"👥 Discovery (L3)",nodes:["niche-discovery-100x","existing-customer-data"]},
  {label:"🎯 Activation (L4)",nodes:["positioning-messaging","pricing-strategy","ecp-validation"]},
  {label:"🎨 Brand (L5)",nodes:["brand-voice","visual-identity"]},
];
```

### Paso 2: Corregir `SKILL_DEPS` — mismos nombres reales

```javascript
const SKILL_DEPS = {
  "company-context": [],
  "business-model-audit": ["company-context"],
  "budget-constraints": ["business-model-audit"],

  "competitor-intelligence": ["company-context","business-model-audit","budget-constraints"],
```

### Paso 3: Corregir `SKILL_CATS.utils` — quitar inexistentes, añadir reales

- **Quitar** `competitor-alternatives` (no existe)
- **Quitar** `nano-banana-pro` (skill de sistema, no de workspace)
- Opcionalmente añadir skills del workspace que faltan en las categorías (ej: `pricing-hooks`, `funnel-architect`, `gtm-orchestrator`, `social-media-extractor`, `apify`, `apollo`, etc.)

### Paso 4: Regenerar `skills-data.js`

El archivo `skills-data.js` solo contiene 38 skills pero hay 62+ directorios. El script de generación (en `mc-server.js` o regenerate) debe incluir todos los skills del workspace para que el panel lateral funcione con cualquier skill.
