# PRD — T-015: Dispatch bot real (discord.js)

## Objetivo

Crear un bot Node.js separado que automatice el dispatch de tareas desde #campaigns a los canales de agentes especializados, creando threads con briefs estructurados al reaccionar con ✅.

## Contexto y motivación

- Actualmente el dispatch de tareas de Sancho a los sub-agentes es manual o via sessions_send.
- El flujo ideal: Sancho publica un brief en #campaigns → Alfonso (o Sancho) reacciona con ✅ → el bot parsea el brief → crea threads en los canales correspondientes (#seo, #paid, #content, etc.) → cada agente recibe su brief específico.
- Esto elimina fricción y hace el workflow auditado (todo queda en Discord).

## Requisitos funcionales

- **RF-1**: Bot discord.js v14 que escucha reacciones ✅ en #campaigns
- **RF-2**: Parsea el mensaje para extraer: tipo de tarea, agente destino, brief, deadline
- **RF-3**: Crea thread en el canal del agente destino con el brief formateado
- **RF-4**: Confirma en #campaigns con reply: "✅ Dispatched to #canal → thread link"
- **RF-5**: Lee `dispatch-map.json` para mapear tipo de tarea → canal destino
- **RF-6**: Logging de todas las acciones dispatch
- **RF-7**: Gestión con PM2 (start, stop, logs, auto-restart)

1. Crear directorio `scripts/dispatch-bot/` con package.json
2. Implementar bot: connection, reaction listener, parser, thread creator
3. Definir formato de brief estándar que Sancho usa en #campaigns
4. Configurar dispatch-map.json con mapeos actualizados (4 agentes, no 12)
5. Configurar PM2 ecosystem file
6. Test end-to-end: brief → reacción → thread
7. Documentar en TOOLS.md

## Criterios de aceptación

## Riesgos y dependencias

| Riesgo | Mitigación |
|--------|-----------|
| Bot necesita token separado o usa el mismo de SanchoCMO | Puede reusar el token existente de SanchoCMO; evaluar si mejor un bot separado |
| Conflicto con OpenClaw Discord plugin | El bot solo escucha reacciones, no mensajes; no debería interferir |
| Formato de brief no estandarizado | Definir template en BRAIN.md que Sancho siga siempre |
