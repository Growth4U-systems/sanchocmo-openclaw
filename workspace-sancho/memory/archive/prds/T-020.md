# PRD — T-020: Backup automático de datos de cliente

## Objetivo

Backup nocturno automático de todos los datos de cliente (brand/, campaigns/, intelligence/, memory/) con retención 30 días y alerta si >48h sin backup.

## Contexto y motivación

- Los datos de cliente (Foundation docs, campañas, intelligence) viven en archivos locales en el workspace.
- No hay backup — un disco corrupto o un `rm` accidental pierde todo.
- Los datos de cliente son el asset más valioso del sistema.

## Requisitos funcionales

- **RF-1**: Backup nocturno (cron a las 03:00 Madrid)
- **RF-2**: Tres opciones de destino (implementar la más viable):
  - **Opción A**: `git init` + commit automático (local + push a repo privado)
  - **Opción B**: `rsync` a directorio externo (otro disco, NAS)
  - **Opción C**: Google Drive via `gog drive upload`
- **RF-3**: Incluir: `brand/`, `campaigns/`, `memory/`, `intelligence/`, `TASKS.md`, `MEMORY.md`, `*.json`
- **RF-4**: Excluir: `node_modules/`, `.cache/`, `*.log`, `skills/` (esas son código, no datos)
- **RF-5**: Retención 30 días (borrar backups >30d)
- **RF-6**: Alerta en #admin si >48h sin backup exitoso
- **RF-7**: Log de cada backup: timestamp, tamaño, archivos, duración

## Archivos afectados

| Archivo | Acción |
|---------|--------|
| `scripts/backup.sh` | Crear — script de backup |
| Cron OpenClaw o LaunchAgent | Crear — schedule nocturno |
| `memory/backup-state.json` | Crear — tracking de backups |

## Pasos de implementación

1. Evaluar mejor opción: git (más simple, versionado) vs rsync vs Drive
2. Implementar script de backup con inclusión/exclusión
3. Configurar cron nocturno (03:00)
4. Implementar retención (borrar >30d)
5. Implementar alerta >48h sin backup
6. Test: ejecutar manualmente, verificar backup, verificar restore

## Criterios de aceptación

- [ ] Backup se ejecuta automáticamente cada noche
- [ ] Incluye todos los datos de cliente, excluye lo innecesario
- [ ] Retención 30 días funciona (backups viejos se borran)
- [ ] Alerta si >48h sin backup
- [ ] Restore probado: se puede recuperar datos de un backup
- [ ] Log de cada ejecución

## Riesgos y dependencias

| Riesgo | Mitigación |
|--------|-----------|
| Git push falla (no hay repo remoto configurado) | Empezar con git local; añadir remote después |
| Backup crece mucho con assets/imágenes | Excluir assets grandes; solo texto/markdown/json |
| Backup de medianoche no corre si Mac está dormido | Usar `launchd` con `StartCalendarInterval` (wakes Mac) o aceptar skip |
