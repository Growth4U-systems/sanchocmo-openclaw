# PRD — T-036: Deep-research como profundizador universal de Foundation

## Objetivo

Crear un patrón transversal "deep-research" que acepta cualquier documento de Foundation como input, lo investiga en profundidad con fuentes verificadas, y devuelve el mismo formato enriquecido. Aplica a TODOS los skills de Foundation.

## Contexto y motivación

Los skills de Foundation (company-context, market-intelligence, competitor-intelligence, etc.) generan documentos rápidos basados en lo que saben + búsquedas ligeras. Esto es suficiente para un primer draft, pero los clientes necesitan profundidad real para tomar decisiones.

**Patrón actual**: Skill genera doc → usuario lo lee → si quiere más, tiene que pedir manualmente y el skill repite todo desde cero.

**Patrón propuesto**: Skill genera doc rápido → lo presenta en hilo de Discord → usuario decide si profundizar → deep-research toma el doc como base, investiga a fondo, y reemplaza con versión enriquecida en el mismo formato.

Esto convierte cada skill de Foundation en un sistema de dos velocidades: fast draft + deep dive on demand.

## Requisitos funcionales

### RF-1: Interface del skill deep-research
- **Input**: Ruta al documento base (`.md`) + template del skill original (opcional)
- **Output**: Mismo documento, mismo formato, con:
  - Más profundidad en cada sección
  - URLs verificadas para cada dato (según T-035)
  - Datos más granulares (desglose por segmento, geografía, etc.)
  - Fuentes cruzadas (mínimo 2 fuentes por afirmación cuantitativa)
- **Mecanismo**: `web_search` + `web_fetch` intensivos (10-20 búsquedas por sección)

### RF-2: Integración con todos los skills de Foundation
Los 16 skills de Foundation deben incluir al final de su ejecución:
```
---
📊 **¿Quieres profundizar?**
Puedo lanzar deep-research para ampliar este análisis con más fuentes, datos más granulares y validación cruzada.
→ Escribe "profundizar" para continuar.
```
Al recibir "profundizar", el skill invoca deep-research pasando el doc recién generado.

### RF-3: Preservación de formato
- Deep-research debe respetar la estructura exacta del template original
- Las secciones del documento base se mantienen — se expanden, no se reemplazan
- Si el template tiene campos específicos (ej: TAM, SAM, SOM en market-intelligence), deep-research los rellena con datos más precisos
- El documento final debe ser drop-in replacement del original

### RF-4: Flujo en Discord (hilos)
1. Skill rápido genera doc → lo presenta en hilo de Discord
2. Usuario lee el resumen en el hilo
3. Usuario escribe "profundizar" (o "deep-research", "más detalle")
4. Sancho invoca deep-research con el doc como base
5. Deep-research investiga (puede tardar 2-5 min)
6. Resultado se publica en el mismo hilo, reemplazando o complementando el original
7. El archivo .md en `brand/{slug}/` se actualiza con la versión profunda

### RF-5: Calidad de investigación
- Mínimo 10 fuentes verificadas por documento profundizado
- Búsquedas en español E inglés (mercados internacionales)
- Priorizar fuentes: informes de consultoras > prensa especializada > publicaciones oficiales > artículos académicos
- Cada fuente debe incluir: título, organización, fecha, URL
- Validación cruzada: si un dato solo aparece en una fuente, marcarlo como `⚠️ Fuente única`

### RF-6: Gestión de versiones
- Antes de reemplazar, guardar backup: `{archivo}.v1.md`
- El documento profundizado se marca con `<!-- deep-research: YYYY-MM-DD -->`
- Log de fuentes consultadas en `brand/{slug}/intelligence/research-log.json`

## Pasos de implementación

1. **Crear skill `deep-research/`** con SKILL.md, templates y references
2. **Definir la interface**: qué recibe (doc path + template) y qué devuelve (doc enriquecido)
3. **Implementar lógica de investigación**: web_search + web_fetch por sección, citación, validación cruzada
4. **Añadir bloque de sugerencia** a los 16 skills de Foundation (batch edit)
5. **Implementar detección de trigger**: "profundizar", "deep-research", "más detalle" → invocar skill
6. **Implementar versionado**: backup de doc original + marca de deep-research
7. **Test con market-intelligence de Hospital Capilar**: generar doc rápido → profundizar → verificar calidad
8. **Test con company-context**: verificar que funciona con otro skill de Foundation

## Criterios de aceptación

- [ ] Skill `deep-research` creada y funcional
- [ ] Acepta cualquier doc de Foundation como input
- [ ] Output respeta formato del template original
- [ ] Mínimo 10 fuentes verificadas en output profundizado
- [ ] URLs realmente visitadas (no inventadas)
- [ ] Los 16 skills de Foundation incluyen sugerencia de profundizar
- [ ] Trigger "profundizar" invoca deep-research correctamente
- [ ] Backup automático del doc original antes de reemplazar
- [ ] Test exitoso con al menos 2 skills diferentes (market-intelligence + company-context)

## Riesgos y dependencias

| Riesgo | Mitigación |
|--------|-----------|
| Tiempo de ejecución largo (2-5 min por doc) | Avisar al usuario: "Investigando... esto puede tardar unos minutos" |
| Alto consumo de tokens (10-20 web_search por sección) | Usar Sonnet para la investigación, Opus solo para síntesis final |
| Fuentes de nicho difíciles de encontrar | Buscar en inglés + español; si no hay datos, documentar gap |
| Formato inconsistente entre skills | Cada skill pasa su template; deep-research lo respeta |
| Depende de T-035 para estándar de citación | Implementar T-035 primero (son complementarios) |

## Estimación de esfuerzo

- **Crear skill deep-research**: 2h (SKILL.md + templates + references + metodología)
- **Editar 16 skills de Foundation** (batch): 1h
- **Testing con 2 skills**: 1h
- **Total**: ~4h
- **Modelo recomendado**: Escudero (Sonnet) para la investigación, Sancho (Opus) para síntesis
- **Orden**: Implementar T-035 primero → luego T-036
