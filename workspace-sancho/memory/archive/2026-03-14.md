## Project Management in Mission Control (Cervantes session, #infra)

### Built: MC Projects UI (`/projects` route)
- **Portal**: `/mc/portal/{token}/projects/` — client-scoped project dashboard
- **Admin**: `/mc/admin/{token}/projects/{slug}/` — admin view per client
- Added nav link "📋 Proyectos" in mission-control.html sidebar

### Two views with tabs
1. **"Por Proyecto"** — expandable project cards with tasks inside, objective, strategy, phase, review date, progress bar
2. **"Por Tarea" (Kanban)** — 4 columns: Por hacer | En progreso | Bloqueado | Completado. Drag & drop (desktop, header-only drag handle). Mobile: dropdown selects for status change.

### Channel mapping (Discord channels of client server)
- Each task has `channel` field mapped to actual Discord channels: web, content, paid-ads, prospecting, partners, creatives, research, brand, intelligence, learning
- Visible as badge on every task in both views

### Edit functionality
- ✏️ button on each task → modal: name, description, channel (dropdown), owner, status
- ✏️ button on each project → modal: name, objective, strategy, status, review date
- Saves to JSON via API

### APIs added to mc-server.js
- `POST /api/projects/task-status` — update task status only
- `POST /api/projects/task-update` — update any task fields (name, description, owner, channel, status)
- `POST /api/projects/project-update` — update project fields (name, objective, status, strategy, review_date)
- `GET /api/projects/{slug}` — get all project data as JSON

### Data model updates (strategic-plan skill)
- `owner` convention: "Sancho" default, "Equipo" for human-required tasks
- `description` field mandatory on all tasks
- `channel` field for Discord channel routing
- Updated `references/data-model.md` with owner/description conventions
