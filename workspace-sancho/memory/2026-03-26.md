# 2026-03-26 — Cervantes Session Notes

## MC Projects UI — Continued Improvements

### Sidebar reorganization
- Moved Idea Bank and Recurrentes from "Overview" to client-scoped section "Trabajo"
- New sidebar structure: Foundation | **Trabajo** (Proyectos, Idea Bank, Recurrentes) | **Datos** (Métricas, Trust Engine, Campañas, Supabase) | Sistema
- Alfonso wants the three work tools grouped together but not merged into one

### Project archiving
- Added 📦 archive button per project with optional reason prompt
- Archive updates: registry.json, project.json, strategic-plan/current.md (appends "Proyectos archivados" section)
- Archived projects hidden from main list + kanban, shown in collapsible "📦 Archivados" section
- API: `POST /api/projects/project-archive` — new endpoint
- Not yet tested by Alfonso ("déjalo pendiente para probar después")

### Notification system (server-side)
- Added `notifyProjectChange()` in mc-server.js — fires on task/project status changes
- Notifies MC Chat thread (via `mcChatAddMessage`) + queues Discord notification in `_system/notification-queue.jsonl`
- Uses `sourceThread` parameter to notify the correct MC Chat thread (not just `:general`)
- All API endpoints updated: task-status, task-update, project-update, project-archive
- Frontend passes `mcChatSidebar._lockedThreadId` as sourceThread in API calls
- Alfonso: "siempre tienes que hacerlo dentro del thread adecuado desde donde hayas escrito"

- Added Rule 15: "Notificar al completar + links" — mention user + MC tokenized links

- Alfonso preferred server-side notifications over SOUL.md rules ("más que ponerlo como regla del Soul MD")

- `execCb` replaced with `spawn` for openclaw agent CLI calls (shell escaping was breaking)

- Click on project now opens full page (not expandable card)

### Alfonso's design philosophy (recurring theme)
- "No inventes cosas nuevas — usa lo que ya existe" (sidebar)

- Prefers server-side guarantees over prompt-based rules
