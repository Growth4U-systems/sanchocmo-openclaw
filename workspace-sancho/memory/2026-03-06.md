## niche-discovery-100x: Major Rewrite to v4.0

### Context
Alfonso ran niche-discovery on Growth4U, got 27 personas × 8 Pain Clusters. Result was solid research but not actionable. Key insight: personas were ARCHETYPES (who you are) not NEEDS (what problem you have).

### Changes Made (in #niche-discovery-skill thread, channel 1478902301182791730)

**v3.3 → v4.0 paradigm shift: Cluster by NEED, not by PERSONA**

1. **Step 3: Solution Filter** (replaces ICP Filter) — Filters by "can we solve this?" not "does this person fit our ICP?"
2. **Step 4: ECP Clustering + Reachability Discovery** — Cluster by SAME CORE NEED + SAME TRIGGER. Reachability is a discovery PROCESS: Trust Map → Search Map → Channel Map (convergence = primary channel)
3. **Step 5: Scoring 2-99** (not 1-10) with mandatory 200-400 char explanations per score. Formula: Pain × 0.35 + Reachability × 0.40 + SAM × 0.25. Founder Moat = badge (🏆/⭐/—), not filter.
4. **Outputs: 2 files** — problems.md (audit trail) + ecps.md (action for downstream skills)
5. **JTBD Synthesis per ECP** — unified_problem + why + jtbd_statement + hypothesis + alternatives (added after first review)
6. **Self-QA checklist** — ECP named by need not persona, channels specific with actions, scoring evidence-based
