## Bugs Corregidos

### Mission Control — Documents Vacío (Bug Crítico)
**Problema:** Desde esta mañana, toda la sección Documents de Mission Control mostraba pantalla en blanco. Ningún archivo ni carpeta visible.

**Root Causes (2):**

1. **Race condition en `renderDocBrowserRoot`** (mission-control.html ~línea 2827)
   - La función construía HTML inicial con Foundation sections
   - Lanzaba `fetch()` asíncrono para cargar carpetas extras del servidor
   - Ejecutaba `el.innerHTML = html` DESPUÉS del fetch → borraba el contenido que el fetch intentaba agregar
   - **Fix:** Mover `el.innerHTML = html` ANTES del fetch, para que `insertAdjacentHTML` pueda agregar contenido después sin ser sobrescrito

2. **CORS headers faltantes** (mc-server.js)
   - El servidor NO enviaba `Access-Control-Allow-Origin: *` en responses HTML
   - Navegador bloqueaba todos los fetches internos desde `sancho-cmo.taild48df2.ts.net` por política CORS
   - Logs mostraban: `net::ERR_FAILED`, `Access-Control-Allow-Origin header is present` (misleading — el header NO estaba presente)
   - **Fix:** Agregado `'Access-Control-Allow-Origin': '*'` a todos los `res.writeHead(200, { 'Content-Type': 'text/html' })` del servidor (13 endpoints)

**Estado:** Ambos fixes aplicados. MC Server reiniciado (PID 42621). Alfonso reportó que sigue vacío — pendiente diagnosticar si Foundation data (`D`) existe para el cliente o si hay problema de CSS ocultando contenido.
