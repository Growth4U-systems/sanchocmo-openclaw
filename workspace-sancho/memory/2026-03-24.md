## MC Chat Sidebar — Reusable Module (Mar 25)

### `mcChatSidebar` Module
- Refactored from metrics-specific chat into a **reusable component** at Alfonso's request
- Any MC page can add a chat sidebar with just: anchor div + toggle button + `mcChatSidebar.init()`
- Mirrors the main dashboard chat (Col 4): same threads, messages, status, typing indicator
- Syncs from main chat every 2s when open
- Auto-closes when user navigates away from the page
- Currently used in the Metrics page

### Usage Pattern (for future pages)
```html
<div id="mc-sidebar-chat-anchor"></div>
<button onclick="mcChatSidebar.toggle()">💬 Chat</button>
```
```js
mcChatSidebar.init({ anchorId: 'mc-sidebar-chat-anchor', toggleBtnId: 'my-btn' });
```

## Dashboard v2 Redesign — Okara-Inspired (Mar 24)

### Design Process
- Alfonso showed Okara AI CMO dashboard as reference — liked the 4-column layout, live terminal, chat
- Did competitive analysis: Okara vs SanchoCMO capabilities. Sancho has much more depth (real APIs, Foundation process, multi-client, projects)
- **9 mockup iterations** (v1-v10) to reach approved design:
  - v1-v3: Comic UI style → Alfonso said "horrible", too cluttered
  - v4: Dark theme → "fondo oscuro no me gusta"
  - v5: Light with Comic borders → "cada columna no está siendo bien separada"
  - v6: Clean SaaS style (Inter font, subtle borders) → first positive reaction
  - v7-v8: Added sidebar, Foundation interactive, thread system in chat
  - v9: Full Foundation with all pillars/syntheses, Activity bar replacing Terminal
  - v10: Thread system in chat (historial + active thread view) → **APPROVED**

### Approved Layout (mockup v10)
- **Sidebar** (210px): brand, client selector, nav links, agent status
- **Activity bar** (collapsible): event feed with timestamps, not a CLI terminal
- **4 equal columns**:
  1. **Brand Foundation**: onboarding URL input + clickable pillar list with states (⬜🔧🟡✅)
  2. **Métricas**: dual mode — PageSpeed (no metrics plan) or real APIs (with metrics plan)
  3. **Próximos pasos**: ⚡Ahora (urgent actions) + 📋Estrategias (with progress) + ❓Decisiones
  4. **Chat**: thread system — historial list + active thread view + new thread button
- **Design system**: Inter font, #F8F8F6 bg, #FFFFFF cards, 1px borders, #C45D35 accent
- Mockup files: `brand/sanchocmo/presentations/dashboard-v2-mockup.html` (with metrics), `dashboard-v2-no-metrics.html`, `dashboard-v2-with-metrics.html`

### Key Design Decisions (Alfonso)
- Foundation is an **onboarding flow** — empty for new clients, URL input first
- Each pilar click → creates/opens a **thread** in the chat column
- Threads are persistent per pilar/project/action — not tabs
- Chat has historial view to browse all threads
- States: pendiente → en progreso → esperando confirmación → confirmado → cerrado
- Métricas without metrics plan: show PageSpeed Insights, Core Web Vitals, SEO Quick Audit
- Métricas with metrics plan: real APIs (GA4, GSC, Meta Ads, GHL, Metricool)
- "Próximos pasos" without Strategic Plan: "Completa Foundation" + quick win ideas with disclaimer
- Activity replaces Terminal — shows what system IS doing, not a CLI aesthetic
- **Responsive required** — mobile must work
