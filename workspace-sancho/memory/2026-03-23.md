# 2026-03-23

## Metrics Plan System — Complete Architecture Built

### Alfonso's Vision (Mar 22-23)
- Dashboard should NOT be generic numbers by integration
- It must be **plan-driven**: the client's metrics plan defines what shows up top
- The plan comes from the `acquisition-metrics-plan` skill during Foundation
- During plan creation, the skill should **ask the client what tools they use** and guide them to connect integrations
- **Plantillas/templates should be pre-defined** in MC so clients can see their archetype's structure
- Growth4U configuration runs from its own guild, not from #infra

### What Was Built

#### 1. `schemas/integration-mappings.json` — Integration Templates
- 12 integrations defined: GA4, GSC, Meta Ads, Google Ads, LinkedIn Ads, GHL, HubSpot, Metricool, Instantly, Shopify, Stripe, Sheets
- Each has `provides` (what metrics it can supply) and `funnelMaps` (how funnel steps map to its API fields)
- Example: funnel step "Meetings" → GHL maps to `appointments`, HubSpot maps to `meetings.count`

#### 2. `scripts/generate-plan.js` — Plan Generator
- Usage: `node generate-plan.js --slug <client> --archetype <type> [--sub-variant <variant>]`
- Reads client's `integrations.json` + `integration-mappings.json`
- Maps funnel steps to connected APIs (or marks as `manual`)
- Calculates which KPIs are possible (cross-source like CPL = ads.spend / crm.newContacts)
- Detects missing integrations
- Outputs `brand/{slug}/metrics-plan.json`
- Tested: Growth4U → 4-step funnel (3 auto, 1 manual), 18 KPIs, 1 missing (Instantly)

#### 3. SKILL.md Updated (acquisition-metrics-plan)
- Added **Step 7.5**: Generate `metrics-plan.json` via `generate-plan.js`
- Added **Step 8**: Integration Discovery Flow — skill asks client what tools they use per metric category, sends MC connect links
- Output section: now 3 deliverables (plan.md + plan.json + XLSX template)
- Context Lake: added `metrics-plan.json` and `integration-mappings.json`

#### 4. Dashboard Plan-Driven Layer
- MC `/api/metrics` now returns `plan` alongside daily data
- Dashboard renders **funnel visualization** above health cards when `metrics-plan.json` exists
- Each funnel step shows: value from API, 🟢 auto / 🟡 manual indicator
- Plan header shows: archetype, activation event, primary KPI, benchmarks
- Missing integrations shown as warning pills
- Cross-source KPIs (CPL, conversion rates) calculated and added to health cards

#### 5. MC Server Updates
- `/api/metrics` response now includes `plan` field
- Also includes archetype templates from `integration-mappings.json`
- Plan card uses `insertAdjacentHTML` with dedup via `#metrics-plan-card` element
