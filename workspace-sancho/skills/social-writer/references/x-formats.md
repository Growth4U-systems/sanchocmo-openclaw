# X / Twitter Formats Reference

## Platform DNA
- 280 chars per tweet, long-form up to 25,000 (sweet spot 1,000-2,000)
- Fast, casual, opinion-driven. Lowercase default.
- You have 0.5 seconds to stop the scroll
- More technical, builder-oriented audience
- Write X FIRST in the repurposing chain (forces conciseness)

## Content Rules
- Use hooks: contrarian and proof hooks perform best
- Line breaks between every thought. Never dense paragraphs.
- No hashtags. Ever. They look spammy.
- Emojis only at the end as signoff (or none at all)
- Links in a REPLY to your post, never in main tweet (algorithm penalizes)
- Single-tweet sweet spot for engagement: **200-240 chars** (leave breathing room, not a wall)

## Formats That Work

### 1. Step-by-Step Thread
"Here's [N] steps to [outcome]:"
- Numbered steps, each tweet = 1 step
- Most reliable format for engagement
- Open loop in first tweet, deliver in thread
- Number explicitly as `x/n` (e.g. `2/7`) — readers track progress; un-numbered threads feel disorienting on long scroll. If your hook tweet starts with `🧵`, you can drop the number on tweet 1.
- **Hook tweet is 80% of the success.** Spend more time on tweet 1 than the rest combined. Reword 5 versions.
- Each tweet must stand alone — readers see them out of order in feeds.
- Max 250 chars per tweet (leave room for the `x/n` numbering).
- Include 1-2 tweets with hard data / stats — they get screenshotted and shared.
- Final tweet: CTA + "Si te fue útil, RT el primer tweet 🔄"

### 2. Short Take
2-4 lines. Bold claim + one-line context + punchline.
- Best for contrarian hooks
- No filler words
- Last line = the punch

### 3. Proof Post
"[Metric] → [metric] in [timeframe]"
- Real numbers that people can't ignore
- Follow with the breakdown (what you did)
- Screenshots/data if available

### 4. Resource Drop
"I just found [thing] — [why it matters]."
- Short, high value
- Always add a personal take (not just sharing)
- Link in reply

### 5. Long-Form Tweet
1,000-2,000 characters. Deep breakdown of one idea.
- Use sparingly (1-2x/week max)
- Structure with line breaks
- Strong hook in first line

### 6. Quote Tweet
Use when there's a relevant tweet/news to react to (often paired with `signal_type` like `conflict`, `contrarian`, or `metric`).

Per quote tweet:
- **Original tweet type**: what is it (e.g. "noticia sobre {tendencia}", "claim del competidor", "dato sectorial")
- **Quote**: max 200 chars — leaves room for the embedded tweet
- **Goal type**: hot-take / complement-data / disagree-constructive

Rules:
- Add a perspective the original didn't have. Don't just nod.
- "Disagree constructive" = name what's wrong AND offer the better take.
- Never QT just to broadcast — that reads as parasocial.

## Strategic Replies (engagement format, not a draft format)
Used when `content_type == "Strategic Replies"` — output 5 ready-to-use replies for niche conversations.

Per reply:
- **Context**: type of tweet to reply to
- **Reply**: max 250 chars, contributes data/experience/perspective
- **Goal**: networking / authority / visibility

Rules:
- Never self-promote. Never link to own content.
- Add a number, a story, or a sharp question.
- One reply = one idea. Don't stack 3 takes in one reply.

## Posting Strategy
- 5-7x/week minimum (1-2 posts per day)
- Best times: 8-9am, 12-1pm, 5-6pm (audience timezone)
- Engage in replies for 30 min after posting (algorithm boost)
