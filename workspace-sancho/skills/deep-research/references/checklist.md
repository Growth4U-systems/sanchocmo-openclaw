# Deep Research — Self-QA Checklist

> Para cada ítem: ✅ | ⚠️ (justificado) | ❌ (seguir investigando)
> Solo entregar con 0 ❌.

## Input
- [ ] **Documento base leído** completamente
- [ ] **Gaps identificados** (secciones sin fuente, datos superficiales)
- [ ] **Estructura original preservada** (drop-in replacement)

## Investigation
- [ ] **Mínimo 10 fuentes verificadas** en el documento final
- [ ] **Búsquedas en español E inglés** realizadas
- [ ] **3-5 web_search por sección** ejecutados
- [ ] **web_fetch** en fuentes más relevantes (lectura completa)
- [ ] **Validación cruzada**: cifras importantes con 2+ fuentes
- [ ] **0 URLs inventadas** — todas visitadas y verificadas

## Output
- [ ] **Estructura original respetada** (secciones no eliminadas/reordenadas)
- [ ] **Datos nuevos añadidos** con fuente inline
- [ ] **Datos sin fuente marcados** con ⚠️
- [ ] **Sección ## Fuentes** al final con lista numerada completa
- [ ] **Marca deep-research** al inicio del documento

## Versionado
- [ ] **Backup realizado** (current.md → v{N+1}.md)
- [ ] **research-log.json** actualizado
- [ ] **Guardado en carpeta del pilar** (no archivo suelto)

## META
- [ ] **Source quality**: No blogs genéricos, no pre-2023, no Wikipedia
- [ ] **Coherencia** con otros brand files
- [ ] **Fuentes contradictorias** presentadas como rango
- [ ] **Gaps restantes** documentados explícitamente
